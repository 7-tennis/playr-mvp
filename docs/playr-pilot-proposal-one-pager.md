# PlayR Pilot Proposal

## Positioning

PlayR is a connected tennis platform that helps players, clubs, and coaches manage ratings, bookings, events, and match activity in one place.

The pilot introduces PlayR as the player-facing experience and ClubR as the club/admin layer, with CoachR planned as a future coaching module.

## The Problem

Many clubs and academies still manage participation through disconnected tools:

- Too much WhatsApp admin.
- Scattered player information.
- Unclear player levels and ratings.
- Manual court booking coordination.
- Event entries handled through messages, spreadsheets, or paper lists.
- Limited visibility for players, coaches, and club admins.

This creates extra admin work and makes it harder for players to understand what to do next.

## The PlayR MVP Solution

The current PlayR MVP connects the main participation loop:

- Player profiles.
- Linked junior profiles.
- Provisional PlayR Rating.
- Court bookings.
- Club events.
- Event entries.
- Match invites and results.
- ClubR admin dashboard.
- Future CoachR direction for coaching feedback and development tracking.

The MVP is designed to test the core flow before adding heavier features like online payments, native apps, or advanced tournament automation.

## Pilot Scope

A pilot club would test a controlled version of the platform by:

- Onboarding a small group of players.
- Creating active courts.
- Creating one or more club events.
- Allowing players to book courts.
- Allowing players to enter events.
- Testing the match invite and result flow.
- Reviewing players, bookings, events, entries, and results through ClubR.

## Suggested Pilot Size And Duration

Recommended starting point:

- 20 to 50 players.
- 1 to 3 club/admin users.
- 2 to 4 courts.
- 1 to 3 events.
- 2 to 4 weeks.

This is large enough to produce useful feedback, but small enough to manage closely.

## Required Setup

Before the pilot starts, prepare:

- Test/pilot club name.
- Court list.
- Admin contact.
- Player test group.
- Event details.
- Booking rules.
- Feedback contact.
- Supabase/test environment if relevant.

## Success Metrics

The pilot should measure:

- Number of player signups.
- Number of completed profiles.
- Number of court bookings made.
- Number of event entries.
- Number of match results submitted.
- Admin time saved or reduced.
- Feedback from players and admins.
- Most requested next feature.

## What Is Included Now

The current MVP includes:

- PlayR player dashboard.
- ClubR admin dashboard.
- Player profiles and junior profiles.
- Club events and event entries.
- Courts and court bookings.
- Match invites and results.
- Ratings and rating changes.

## What Is Not Included Yet

The MVP does not yet include:

- Online payments.
- Full CoachR schedule and feedback module.
- Native iOS or Android app.
- Full multi-club permissions.
- Automated notifications.
- Advanced tournament automation.

These are intentionally left for later so the pilot can validate the core participation workflow first.

## Future Roadmap

Future areas may include:

- CoachR for schedules, feedback, lesson notes, and development tracking.
- Online payments.
- Notifications.
- Native mobile app.
- Richer club and member management.
- Advanced events and tournament formats.
- Multi-club support.

## Pilot Responsibilities

### PlayR Responsibilities

- Setup support.
- Demo and training.
- Pilot adjustments where appropriate.
- Feedback collection.
- Help reviewing pilot results.

### Club Responsibilities

- Provide test users.
- Provide courts and event data.
- Appoint an admin contact.
- Test real player and admin flows.
- Give feedback during and after the pilot.

## Next Steps

1. Confirm pilot interest.
2. Choose pilot dates.
3. Prepare test data.
4. Create ClubR admin account.
5. Onboard player test group.
6. Run first demo.
7. Collect feedback and review results.

## Closing

The goal of the pilot is not to replace every club process immediately, but to prove that PlayR can reduce admin, improve player visibility, and create a better connected tennis experience.
